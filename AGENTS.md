# AGENTS.md

本仓库用于使用纯 Markdown 文件系统维护 wiki。

## 1. 目标

- 将原始资料整理为结构化、可查询、可持续维护的 wiki。
- 查询时只基于 wiki 回答，不回查原始资料。
- 尽可能保留原文中的关键信息，不做过度摘要。

## 2. 仓库结构

- `raw/`：原始资料，只读，不修改。
- `wiki/`：整理后的 wiki 页面，由 agent 维护。
- `wiki-拆分指导.md`：拆分规范文件时必须遵循的指导文档。
- `wiki-查询指导.md`：查询 wiki 时必须遵循的指导文档。
- `常用提示词汇总.md`：可复制的一行/短段任务指令（细则仍以各指导为准）。

## 3. 工作方式

- 导入新文档时，先读取原始资料，再按规则拆分到 `wiki/`。
- 拆分、生成 `-index.md`、补充关联文件、生成后检查时，参考 `wiki-拆分指导.md`。
- 查询 wiki 时，参考 `wiki-查询指导.md`。
- 如果需要补充或修正规则，优先更新指导文档，再执行批量处理。

## 4. 基本要求

- 原始资料是事实源，wiki 是查询层。
- 查询阶段不能读取 `raw/` 中的原始文档。
- 输出默认使用 Markdown。
- 对用户简洁汇报当前动作和结果。
