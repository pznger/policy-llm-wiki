# 任务
根据 `correct-answers.md` 和 `answer-x.md`，评估 `answer-x.md` 是否正确，生成 `evaluation-x.md`。

# 评估顺序（先溯源、后结论）

1. **打开 `answer-x.md` 的「关键溯源条文」表格**
   - 是否使用 Markdown 表（含表头：序号、规范、条号/章节、页码、wiki 路径、原文摘抄）？
   - **页码**是否与对应 `*-index.md` 的 `## 条目录` 一致（不得仅凭正文 `<!-- 第 N 页 -->` 判断）？
   - **每一行**是否路径存在、条号与摘抄一致、摘抄在对应 wiki 文件中 **逐字可查**（允许中间 `……`，不得改写数字、主体、否定词）？
2. **再读「答案」**
   - 答案中的时限、倍数、义务主体、条号引用，是否都能在上面的摘抄中找到依据？
   - 是否存在 **wiki 中无对应正文** 却被写成确定结论的情况？→ 记为「编造」。
   - **归纳质量**（溯源无误时）：是否先回应题目、按主题合并（见 `wiki-查询指导.md` §6.4）？若仅为溯源表逐行 bullet 复述、几乎无合并归纳，在评测备注中记「答案罗列，建议重写」，**不单独因此判错**（除非因此漏答子问或歪曲含义）。
3. **与 `correct-answers.md` 比对**（若有）
   - 客观差异：条号错误、数值错误、漏项、跨规范混淆、转引条次与 wiki 正文不符仍当作处罚依据等。

# 评估文件示例

## 回答正确
```
正确
```

## 回答错误
```
错误

错误原因
1. 溯源：answer-x 写「《条例》第六十六条，5～15 倍罚款」，但 wiki 中第六十六条正文为再评价义务，罚款在第八十六条；摘抄缺失或与路径不符。
2. 结论：……（与 correct-answers 或 wiki 原文的差异）
```

# 判定规则

| 情况 | 判定 |
| --- | --- |
| 结论正确且每条有可查摘抄 | 正确 |
| 结论正确但无摘抄、仅条号 | **错误（溯源不足）** |
| 摘抄与 wiki 原文不一致 | **错误（摘抄错误/幻觉）** |
| wiki 无依据却给出确定结论 | **错误（编造）** |
| 转引条次与 wiki 被引规范正文不符且未说明 | **错误（条次误用）** |
| 措辞不同、顺序不同但义务与数值均对且有摘抄 | 正确 |

# 注意事项
- 只比对可客观核实的内容；解释性归纳若不改变法律含义，不算错。
- 评估时 **只读 `wiki/`** 核实摘抄，不读 `raw/`。
