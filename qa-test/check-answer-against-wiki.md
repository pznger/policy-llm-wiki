# 任务
不参考 `correct-answers.md`，仅根据 **`wiki/`** 核对指定答题文件（如 `2026-05-28/answer.md`）是否可溯源、是否存在幻觉。

# 步骤
1. 列出该答案中 **每一条** 带条号的断言（义务、时限、倍数、主体）。
2. 对「关键溯源条文」**表格逐行**检查（须为 Markdown 表，一行一条）：
   - 路径是否存在；
   - 摘抄是否在对应文件中 **逐字出现**（或仅省略中间无关句）；
   - 条号是否与摘抄段落一致；
   - **页码**是否与同目录 `*-index.md` 的 `## 条目录` 中该条号的 `（第N页）` 一致（常见错误：把正文 `<!-- 第 N 页 -->` 当成「在第 N 页」）。
3. 对「答案」中 **无摘抄支撑** 的断言，在 wiki 中 `rg` 检索；仍找不到则记为 **编造或溯源缺失**。
4. 对 **转引他法** 的处罚/义务（如《办法》→《条例》），分别核对两部规范 wiki 正文；若条次内容与转引不符，记为 **条次误用**。
5. 输出 `check-<原文件名>.md`：按「正确 / 溯源不足 / 摘抄错误 / 编造 / 条次误用」分类列表。

# 输出格式示例

```markdown
# 核对结论：<answer 文件路径>

## 总体
可溯源 / 部分可溯源 / 不可接受（编造或大量条次误用）

## 逐条
| 答案中的结论 | 声称条号 | wiki 路径 | 结果 | 说明 |
| --- | --- | --- | --- | --- |
| … | … | … | 通过 | 摘抄匹配 |
| … | 条例第六十六条 5～15 倍罚款 | … | 条次误用 | wiki 第六十六条为再评价，非罚款 |

## 建议修改
……
```

# 注意事项
- 只读 `wiki/`。
- 若 wiki 该规范大量 `TODO: 待LLM补写`，应在结论中说明「评测受限，需先补全 wiki」。
