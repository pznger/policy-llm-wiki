# Wiki 查询指导

回答与政策法规相关的问题时，按本指导操作。**所有查询只允许访问 `wiki/` 下的文件**，禁止读取 `data/`、`raw-0/`、`clean/` 或 `raw/`。

## 1.核心原则

渐进式披露原则。

先读 index，再读完整内容，渐进式读取上下文，防止 token 撑爆上下文。

## 2.查询方式

查询 wiki 时，应遵循以下顺序：

1. 先读取根目录或章节目录下的 `index.md`
2. 根据 `index.md` 判断最相关的章节文件
3. 再读取对应章节的完整内容
4. 如果当前内容仍不足以回答问题，再继续读取关联文件或下一级相关文件

不要在一开始就一次性读取大量正文文件，而是应先通过 `index.md` 缩小范围，再逐步展开。

这样做的目的是：

- 优先获取全局结构
- 控制单次读取的上下文规模
- 避免无关内容占用 token
- 在尽量少读取文件的前提下完成回答

## 3. 回答要求

- **只基于 wiki/ 内容回答**：不要凭"通识"补充未在 wiki 中出现的条款编号或时限。
- **先溯源、后结论**：必须先完成 **「关键溯源条文」Markdown 表格**，再写归纳答案（表格 §6.2，答复 §6.4；`qa-test/question-template.md`）。
- **逐字引用关键信息**：条款编号（`第八十七条`）、时间（`60 日内`）、阈值（`第二、三类`）、法律依据（`《医疗器械监督管理条例》`）等必须按 wiki 原文给出，避免改写。
- **保留溯源**：每条结论在溯源块中应有对应摘抄；答案段可再简写出处，例如：

  ```
  - 持有人应当在每满 1 年后的 60 日内完成上年度产品定期风险评价报告
    —— 《医疗器械定期风险评价报告撰写规范》2.1.1（第xx页）
    wiki/医疗器械定期风险评价报告撰写规范/…/xxx.md
  ```

- **跨规范综合**：先把每本规范的相关条款逐一列出（含摘抄），再做综合判断；不要混搭。
- **转引他法**：若《办法》写「按《条例》第×条处罚」，须同时摘抄《办法》转引句与《条例》该条在 wiki 中的**实际正文**；修订后条次与转引不符时必须说明，不得按条号臆测处罚内容。
- **遇到表格**：直接在回答中嵌入 wiki 中的 HTML `<table>`，或者只引用关键单元格内容并标注来源表格编号。
- **拿不准**：明确说明 wiki 中查不到对应条款，**不要编造**。

## 4. 工作流（典型查询）

```text
用户提问
  └─ 读 wiki/ 下相关规范的 <规范名>-index.md
       └─ 选出 1～3 个可能相关的章
           └─ 读各章中相关节的 -index.md
               └─ 打开命中的正文 .md，提取条款 / 表格 / 定义
                   └─ 综合多个文件给出答案 + 溯源
```

## 5. 体检 / Lint（定期执行）

定期让 LLM 给 wiki 做一次健康检查，检查项包括：

- **重复**：多个文件描述同一条款 / 同一表格。
- **失效链接**：`## 关联文件` 中的相对路径无法解析。
- **缺出处**：正文文件中提到"详见 …" 或"按附表 …" 但 wiki 中找不到对应文件。
- **未补全的索引**：仍存留 `TODO: 待LLM补写`；或「条文摘要」合并多条、仅空泛一句。
- **条目录异常**：出现长段条文截断（应只有 `第十三条（第4页）` 这种格式）。
- **未按页拆分**：单正文文件跨多页却未拆成 `-1`、`-2`。
- **遗漏条款**：某条款在多个上下文中被多次引用，但 wiki 中没有该条款独立片段。
- **格式异常**：表格、HTML 注释、公式被截断；正文文件首行不是标题；中文之间被插入空格。
- **关联缺失**：明显相关的两个文件之间没有 `## 关联文件` 链接。
- **页码归零**：拆分时遗漏页码标记导致溯源失效。

发现问题时先做最小修复（例如补摘要、补关联、修链接），如果是规则层面的问题，**回头更新 `wiki-拆分指导.md` 或脚本**，再批量重跑。

## 6. 答题输出格式（qa-test / 用户问答）

为减少幻觉，**问答与评测**采用固定结构（见 `qa-test/question-template.md`、`qa-test/answer-example.md`）。

### 6.1 文档结构

```markdown
## 问题
（题目）

## 关键溯源条文

（Markdown 表格，见 6.2）

## 答案

（面向客户的归纳答复，见 §6.4；结论须能对应表中序号）
```

### 6.2 关键溯源条文 · 表格格式（必须）

**一节一表**；跨多部规范时可分 `###` 小标题，每标题下各一张表。

| 序号 | 规范 | 条号/章节 | 页码 | wiki 路径 | 原文摘抄 |
| ---: | --- | --- | ---: | --- | --- |
| 1 | 《医疗器械使用质量监督管理办法》 | 第四条 | 1 | `wiki/医疗器械使用质量监督管理办法/第一章总则/总则.md` | 医疗器械使用单位应当按照本办法，配备与其规模相适应的医疗器械质量管理机构或者质量管理人员…… |

| 列 | 要求 |
| --- | --- |
| **序号** | 从 1 递增；答案可写「见溯源表序号 3」 |
| **规范** | 规范全称，带书名号 |
| **条号/章节** | `第四条`、`2.1.1` 等 |
| **页码** | 整数；**必须优先**从该条文所在文件的 **`-index.md` → `## 条目录`** 中对应条号的 `（第N页）` **原样抄写**，不得心算、不得按正文顺序猜页。仅当条目录无该条时，才按正文 `<!-- 第 N 页 -->` 推算（见下）；仍无法确定写 `—` |
| **页码（正文标记语义）** | `<!-- 第 N 页 -->` 表示 **第 N 页已结束**，**其后**正文属于 **第 N+1 页**（与 `scripts/split_wiki.py`、OCR 一致）。**禁止**把标记理解为「本条在第 N 页」——否则会把紧挨标记后的条文（如第九条）错写成上一页。例：`采购、验收与贮存.md` 中 `<!-- 第 2 页 -->` 在第八条中段，第九条在标记之后 → 条目录为 **第 3 页**，不是第 2 页。 |
| **wiki 路径** | 仓库相对路径，反引号包裹 |
| **原文摘抄** | 逐字摘自 wiki；过长可删中间用「……」；**数值、主体、否定词**不得改；单元格内勿用 `\|` |

可选列 **备注**（转引说明、仅 index 等）。

**禁止**：用无序列表代替表格；一行合并多条；只有条号无摘抄。

### 6.3 硬性规则

1. **无摘抄，不写结论**。
2. **摘抄优先读正文 `.md`**；`TODO: 待LLM补写` 时须改读正文。
3. **页码以 index 为准**：填表前对该条打开同目录 `*-index.md`，在 `## 条目录` 找到 `第九条（第3页）` 等形式，**页码列只填数字 `3`**。同一 `.md` 内多条若页码相同或递增，仍须 **逐条查 index**，不得因「都在同一文件」而统一填同一页。
4. **转引他法**：转引句与被引规范正文各占至少一行；条次与 wiki 不符须在备注说明。
5. **评测**时按表逐行核对 `qa-test/evaluate.md`、`qa-test/check-answer-against-wiki.md`。

### 6.4 「答案」归纳要求（面向客户，非罗列条文）

溯源表是**证据层**，「答案」是**答复层**：写给提问者看，须**回答问题**，而不是把表中条文改写成 bullets 再贴一遍。

**必须做到**

1. **先答所问**：开头用 1～2 句直接回应题目（或各子问）；多子问按题号分段，每段先结论后展开。
2. **按主题归纳**：按业务环节、义务类型、主体职责等**合并**溯源表中的多条（如采购、贮存、不良事件各成一段）；**不要**按规范名称或表序号顺序逐条复述。
3. **合并同义义务**：多部规范要求相近时，写成**一条**完整表述，括号注明「溯源表序号 3、18」；避免「《办法》说…《条例》又说…」式并列堆砌。
4. **保留关键要素**：时限、倍数、保存期限、禁止性用语、提交渠道等须写清，数值与主体与摘抄一致；可概括修饰语，**不得**改数字或否定词。
5. **标注出处**：每条归纳末尾用「（序号 n）」或「（序号 n、m）」对应溯源表；无表行支撑不写。

**建议做到**

- 开放式列举题（如「有哪些义务」）：用少量 **二级标题**（3～7 类为宜）概括，每类 2～5 句，不要一类下挂十几条碎 bullet。
- 比较/区分题：可用简短对比句或小型 Markdown 表（仍须在表外或句末标序号）。
- 情景计算题：先给**具体日期/结论**，再括号说明推算依据（序号）。

**禁止**

- 把溯源表每行翻译成一条 bullet（「罗列式答案」）。
- 答案篇幅长于溯源表摘抄之和、却未增加归纳信息。
- 为凑结构编造表中不存在的义务或条号。

**与溯源表的分工**

| 溯源表 | 答案 |
| --- | --- |
| 逐条、逐字摘抄 | 合并、归纳、按问题重组 |
| 一行一条文 | 一段可对应多序号 |
| 供核对与评测 | 供客户阅读与决策 |
